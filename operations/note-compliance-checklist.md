# note投稿コンプライアンスチェックリスト

note公式は、機械的な大量登録・大量投稿、スパム投稿、外部サービスを利用した組織的な悪用・迷惑行為への対応強化を発表している。

このため、本AI会社組織では「AIで量産して投稿する」運用を禁止し、AIは下書き・調査・構成補助に限定する。

## 基本方針

- AIは記事の代筆者ではなく、編集補助・調査補助として使う。
- note投稿は量より質を優先する。
- 社長の一次情報、体験、判断、具体例を必ず入れる。
- 記事は「一次性・独自性・検証可能性」を満たすように設計する。
- 事実、意見、体験談、引用を明確に分ける。
- 公開前に社長が目視で確認し、必要に応じてリライトする。
- 公開ボタンは社長承認後のみ押す。

## 禁止事項

- AI生成記事を未編集のまま投稿する。
- 短時間に大量の記事を機械的に投稿する。
- 同じ構成・同じ言い回しの記事を量産する。
- 競合記事を要約・言い換えただけの記事を投稿する。
- スキ、フォロー、コメント、投稿を外部ツールで機械的に増やす。
- 実体験がないのに体験談風に書く。
- 架空ケースを実体験のように見せる。
- 仮説を事実のように断定する。
- 読者の悩みが不明確な記事を投稿する。
- 読後の次アクションがない記事を投稿する。
- 「やばい」「終わり」「知らないと損」など、煽りだけで読ませる。
- 他人のバズ記事を要約・再構成しただけで、自分の検証や意見がない。
- 出典が追えない情報、疑似科学、フェイクニュースを断定する。
- 「絶対」「確実に治る」「誰でも稼げる」などの強い断定を中心にする。

## 推されやすい記事の条件

noteでは、以下のような記事が評価されやすい。

- 人間による生の一次情報・体験の記録・作品。
- 自身の一次情報を、AIをツールとして活用して編集・構成したもの。
- 実体験、実験レポート、失敗談からの学び。
- 具体的なケーススタディ。
- 時系列や体系的な整理。
- 継続的な体験・実験の記録。
- 作者固有の観察、表現、経験の共有。
- 引用元が明確で、議論が前に進む建設的な引用。
- 記事内で内容が自己完結しているもの。

## 推されにくい記事の条件

- 攻撃、断罪、晒し、嘲笑、ゴシップが中心の記事。
- 不安を煽ることが主目的の記事。
- 過剰な一般化で既存コンテンツを再生産する記事。
- 他人の意見やバズ記事をそのまま再出力した記事。
- 一次情報や検証がなく、調べた内容の浅いまとめで終わる記事。
- 出典が追えない、引用が不正確、検証できない記事。

## 投稿前チェック

| チェック項目 | 判定 |
| --- | --- |
| 誰の悩みを解決する記事か明確か | OK / NG |
| 社長の一次情報・経験・具体例が入っているか | OK / NG |
| 実体験がない部分は、仮説・検証ログ・架空ケースとして明示されているか | OK / NG |
| AIっぽい一般論だけで終わっていないか | OK / NG |
| 競合記事の焼き直しになっていないか | OK / NG |
| 誇大表現・収益保証・成果保証がないか | OK / NG |
| 読後に取るべき次アクションがあるか | OK / NG |
| 投稿頻度が機械的・大量投稿に見えないか | OK / NG |
| 社長が最終リライト済みか | OK / NG |
| 事実と意見が区別されているか | OK / NG |
| 必要な引用元・出典が追えるか | OK / NG |
| note内で内容が自己完結しているか | OK / NG |
| 攻撃・断罪・不安煽りが主目的になっていないか | OK / NG |

1つでもNGがある場合、公開しない。

## Note記事の推奨構成

```markdown
# タイトル

## この記事は誰向けか

## 読者の悩み

## 社長の一次情報・体験

## その体験から得た気づき

## 具体的な解決策

## 実際にやる手順

## 注意点・失敗例

## 参考情報・引用元

## 次にやること
```

## 制作部への追加ルール

- Noteドラフトには「一次情報を入れる場所」を必ず作る。
- AIだけで埋められない箇所は `[社長が追記]` と明記する。
- 競合リサーチは参考に留め、本文の主張は社長の経験・判断・検証に寄せる。
- 1日複数本投稿を前提にした量産計画を出さない。
- 事実、意見、体験談、引用を分けて書く。
- 記事の結論をnote内で完結させ、重要部分を外部LINEなどに隠さない。

## 営業部への追加ルール

- noteへの自動投稿は原則行わない。
- 営業部は投稿前チェックリスト作成、整形、予約投稿準備までを担当する。
- 公開操作は社長の明示承認後に限る。
- フォロー、スキ、コメントの自動化は行わない。
