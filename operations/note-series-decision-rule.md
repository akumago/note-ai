# noteシリーズ化・撤退判断ルール

## 基本方針

note運用では、バズるネタを単発で消費せず、反応がよいテーマをシリーズ化する。  
一方で、閲覧・スキ・コメント・購入などの反応が落ちたテーマは、早めに撤退する。

## 判断する指標

毎週、以下を確認する。

- ビュー数
- スキ数
- コメント数
- フォロー増加
- 有料記事の購入数
- プロフィール遷移
- SNSでの反応
- 保存版や続編希望のコメント

## シリーズ化する条件

以下のうち複数を満たす記事はシリーズ化候補にする。

| 条件 | 内容 |
| --- | --- |
| ビュー | 同期間の平均より明らかに多い |
| スキ率 | ビューに対してスキが多い |
| コメント | 質問・相談・続き希望がある |
| フォロー | 記事公開後にフォローが増える |
| 購入 | 関連する有料記事・テンプレが売れる |
| SNS | Xなどで引用・感想・保存がある |
| 深掘り余地 | 追加で3本以上書ける切り口がある |

## シリーズ化の型

反応がよかった記事は、以下の方向に展開する。

```text
第1回: 問題提起・体験談
第2回: 失敗談・やってはいけないこと
第3回: 実際の手順
第4回: テンプレート・チェックリスト
第5回: 読者質問への回答
第6回: 有料版・詳細版・実例集
```

## 撤退する条件

以下が続く場合は撤退または保留にする。

- 3本出してもビューが伸びない。
- スキ率が低い。
- コメントや質問がない。
- フォロー増加につながらない。
- 有料導線に反応がない。
- 社長の一次情報が尽きた。
- 続けるほど一般論になっていく。

## 撤退時の処理

撤退するテーマは完全に捨てず、以下のいずれかにする。

- 別テーマの記事内の一部として再利用する。
- WordPress保存版に回す。
- 無料記事として残し、導線だけ整理する。
- 3ヶ月後に再調査する。

## 週次編集会議

毎週1回、リサーチ部・制作部・社長で以下を確認する。

```markdown
# 週次note編集会議

## 今週出した記事

## 反応がよかった記事

## 反応が悪かった記事

## シリーズ化候補

## 撤退候補

## 来週すぐ乗るバズネタ

## 有料化候補

## 社長の判断
- 継続:
- シリーズ化:
- 有料化:
- 撤退:
```

## 判断の注意

- 1本だけで即撤退しない。
- ただし、明らかに反応がないテーマを惰性で続けない。
- ビューだけでなく、スキ率・コメント・購入意欲を見る。
- バズったテーマでも、社長の一次情報がないなら深追いしない。
- シリーズ化しても、毎回同じ話を繰り返さない。
