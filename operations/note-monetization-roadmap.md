# note収益化ロードマップ

確認日: 2026-05-29

このロードマップは、AI会社組織を使ってnote運用を行うための収益化設計である。前提として、AIによる機械的な大量投稿ではなく、社長の一次情報・体験・検証結果を中心にした記事制作を行う。

## 1. noteの収益化手段

| 手段 | 内容 | 向いている用途 | 初期導入 |
| --- | --- | --- | --- |
| 有料記事 | 1記事ごとに価格を設定して販売する | 具体的なノウハウ、体験談、テンプレート | Phase 2以降 |
| 有料マガジン | 複数記事をまとめて販売する | 連載、体系化された教材 | Phase 3以降 |
| 定期購読マガジン | 月額で継続記事を配信する | 継続レポート、毎月更新の教材 | Phase 4以降 |
| メンバーシップ | 月額コミュニティを運営する | ファン化、交流、限定コンテンツ | Phase 4以降 |
| チップ / サポート | 読者から応援を受ける | 無料記事の支援 | Phase 1から可 |
| Amazonアソシエイト | 商品紹介による紹介料 | 書籍、道具、ガジェットレビュー | Phase 2以降 |
| AI学習対価還元 | 公開テキスト記事がAI学習に使われた場合の還元 | 良質な公開記事の追加収益 | 設定確認後 |

## 2. 収益化の基本方針

- 最初から有料記事を売り込まない。
- まず無料記事で信頼、代表作、プロフィールを整える。
- 広く届けたい内容は無料にする。
- 狭く深く届けたい内容を有料にする。
- 有料部分は、無料部分より明確に深く、購入者の期待を上回る内容にする。
- 売上公開や過度な煽りで購入を急がせない。

## 3. 5ステップ運用

### Step 1: ジャンル選定

需要がある領域と、社長自身の一次情報が出せる領域の重なりを選ぶ。

候補例:

- 副業 × 自分の失敗談
- キャリア × 年代 / 職種 / 未経験転職
- 子育て × 在宅副業
- 健康 / 美容 × 自分の実体験
- エンジニア / AI × 実際の構築記録
- 文章術 × 自分の記事改善ログ

避けること:

- 稼げそうという理由だけで、経験のないジャンルを選ぶ。
- 需要が小さすぎるテーマまで絞りすぎる。
- 専門性が必要な領域で断定的な助言をする。

### Step 2: 無料記事で信頼を作る

最初に用意する記事:

1. 自己紹介記事
2. 代表作になる無料記事
3. 一次情報・失敗談・検証結果を含む記事

目安:

- 投稿頻度は週1-2本を基本にする。
- 品質が落ちるなら投稿頻度を下げる。
- 最低3ヶ月は継続する。

### Step 3: SNSで外部流入を作る

noteだけに依存せず、X、YouTube、Instagramなどから流入を作る。

媒体ごとの使い方:

- X: note記事の要点、制作過程、読者との会話。
- YouTube: 話した内容をnoteで再構成し、補足情報を加える。
- Instagram: 美容、料理、インテリア、ビジュアル系テーマに向く。

禁止:

- スキ、フォロー、コメントの自動化。
- 同じ誘導文の大量投稿。
- noteの重要部分をすべて外部LINEへ隠す導線。

### Step 4: 有料コンテンツを投入する

無料と有料の切り分け:

| 無料に向く内容 | 有料に向く内容 |
| --- | --- |
| 広く知ってほしい基本ノウハウ | 個別性が高い実例 |
| 認知拡大につながる代表作 | 生々しい失敗談や詳細な数字 |
| 読者の信頼を作る体験談 | テンプレート、チェックリスト、手順書 |
| 検索・SNSで広がってほしい記事 | 特定の読者だけに深く刺さる内容 |

有料化の条件:

- 無料記事で一定の反応がある。
- 読者の悩みが明確である。
- 有料部分の価値が明確に説明できる。
- 誇大表現、収益保証、成果保証をしていない。

### Step 5: 分析と改善

週1回、以下を見る。

- ビュー数
- スキ数
- コメント
- フォロー増加
- 購入数
- 購入導線になった記事
- SNSからの反応

改善の切り分け:

| 状況 | 見るポイント |
| --- | --- |
| ビューが少ない | タイトル、テーマ、SNS導線、投稿時間 |
| スキが少ない | 読者の悩み、一次情報、読みやすさ |
| 購入されない | 無料記事との信頼関係、有料部分の説明、価格 |
| フォローされない | プロフィール、発信テーマの一貫性 |

反応がよい記事はシリーズ化し、反応が落ちたテーマは撤退または保留にする。  
判断は `operations/note-series-decision-rule.md` と `operations/weekly-editorial-meeting-template.md` に従う。

## 4. AI会社組織での役割分担

| 部署 | 担当 |
| --- | --- |
| リサーチ部 | 需要、競合、読者の悩み、参考記事の調査 |
| 制作部 | 無料記事、有料記事、SNS投稿案のドラフト作成 |
| 営業部 | 投稿前チェック、SNS告知文、導線確認 |
| CS部 | 購入者対応、質問一次返信、特典配布 |
| 社長秘書部 | 読者Q&A、よくある質問整理、相談導線 |
| 社長 | 一次情報の追記、最終リライト、公開承認、価格判断 |

## 5. 有料記事の安全チェック

公開前に確認すること:

- 「必ず儲かる」「絶対成功する」などの表現がない。
- 金融商品、医療、法律、税務などの専門助言を断定していない。
- 売上公開で購入を煽っていない。
- 自己購入や不自然な購入誘導をしていない。
- 無料部分で有料部分の内容・対象者・得られるものを誠実に説明している。
- 購入者が期待する内容を有料部分で満たしている。

## 6. 手数料メモ

note公式ヘルプでは、販売時に以下が差し引かれる。

- 事務手数料: 購入者の決済手段により異なる。
- プラットフォーム利用料: 有料記事、有料マガジン、チップ、メンバーシップは10%。定期購読マガジンは20%。
- 振込手数料: 売上金の支払い1回あたり270円。

手数料や仕様は変更される可能性があるため、有料化前に公式ヘルプを確認する。

## 7. AI学習対価還元の扱い

note公式ヘルプによると、公開状態のテキスト記事は原則としてAI学習の対象になる。下書き記事は対象外。希望しない場合は、アカウント全体または記事ごとに提供を許可しない設定ができる。

本システムでの扱い:

- AI学習対価還元は主収益ではなく、副次収益として扱う。
- 個人情報、機密情報、未公開ノウハウを公開記事に含めない。
- AI学習に提供したくない記事は、公開前に設定を確認する。
- 収益見込みを断定しない。

## 8. 最初の90日ロードマップ

| 期間 | やること | ゴール |
| --- | --- | --- |
| Day 1-7 | ジャンル選定、プロフィール、自己紹介記事 | 何者か伝わる状態 |
| Day 8-30 | 代表作無料記事、週1-2本投稿 | 信頼を作る |
| Day 31-60 | SNS連携、反応分析、記事改善 | 読まれる型を見つける |
| Day 61-90 | 小さな有料記事またはテンプレ販売 | 初回収益化を試す |

## 9. 成功条件

- 週1回以上、一次情報を含む記事を出せている。
- 代表作記事が1本ある。
- プロフィールから発信テーマが伝わる。
- 読者の反応を週1回見直している。
- 有料記事を出す前に無料記事で信頼を作れている。
- AIによる量産ではなく、社長の経験を編集する運用になっている。
