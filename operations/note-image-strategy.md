# note画像・図解戦略

確認日: 2026-05-29

## 基本方針

note記事は本文だけでなく、見出し画像・図解・チェックリスト画像・テンプレート画像を組み合わせる。

特に有料記事では、画像や図解があることで以下の効果がある。

- 商品感が出る。
- 読者が内容を理解しやすい。
- SNSでシェアされやすい。
- 有料部分の満足度が上がる。
- テンプレートや手順の価値が伝わりやすい。

ただし、画像もAIっぽい装飾だけにしない。記事の理解を助けるものに限定する。

## note画像仕様メモ

note公式ヘルプでは、登録画像の容量は最大10MBとされている。  
記事の見出し画像などは、公式ヘルプの推奨サイズを確認して作成する。

運用ルール:

- 画像は10MB未満にする。
- 見出し画像はnote公式の推奨サイズに合わせる。
- スマホで文字が読めるサイズにする。
- 文字を詰め込みすぎない。
- AI生成画像を使う場合も、記事内容と関係があるものにする。

## 記事ごとの画像セット

### 無料記事

最低限:

- 見出し画像 1枚
- 記事内図解 1枚

推奨:

- 見出し画像 1枚
- 記事内図解 1-2枚
- 最後に「次にやること」チェックリスト画像 1枚

### 有料記事

最低限:

- 見出し画像 1枚
- 記事内図解 2-3枚
- テンプレート画像 1枚

推奨:

- 見出し画像 1枚
- 全体ロードマップ図 1枚
- 手順図 1枚
- チェックリスト画像 1枚
- 有料特典テンプレート画像 1-3枚

## 画像の種類

| 種類 | 用途 |
| --- | --- |
| 見出し画像 | 記事の第一印象、SNSシェア時の印象 |
| ロードマップ図 | 全体像を一目で見せる |
| フローチャート | 手順や判断分岐を説明する |
| チェックリスト画像 | 読者が保存しやすい形にする |
| 比較表画像 | note / WordPress、有料 / 無料などを比較する |
| テンプレート画像 | 有料記事の商品価値を見せる |
| ケーススタディ図 | 架空ケースや読者像を整理する |

## AI画像生成の使い方

OpenAIの画像生成では、GPT Image系モデルを使って画像生成・編集ができる。  
記事制作では、以下の順番で使う。

1. 制作部が記事の要点を抽出する。
2. 画像担当が必要な画像リストを作る。
3. 見出し画像・図解・チェックリスト画像のプロンプトを作る。
4. 画像を生成する。
5. 文字が読めるか、内容と合っているか確認する。
6. 必要なら画像を修正する。
7. noteへアップロードする。

## 画像生成プロンプトの型

### 見出し画像

```text
note記事の見出し画像。テーマは「[記事テーマ]」。
読者は「[想定読者]」。
印象は、知的、実務的、信頼感、落ち着いた雰囲気。
AI感の強い未来的な装飾は避ける。
文字は入れない。
横長のWeb記事用アイキャッチ。
```

### 図解画像

```text
note記事内に入れる図解画像。
テーマは「[説明したい概念]」。
構成は、左から右へ流れるシンプルなフローチャート。
項目は以下:
1. [項目1]
2. [項目2]
3. [項目3]
4. [項目4]
白背景、読みやすい文字、余白多め、ビジネス記事向け。
```

### チェックリスト画像

```text
note記事の保存用チェックリスト画像。
テーマは「[チェックリスト名]」。
項目は5-7個。
スマホで読みやすい文字サイズ。
白背景、黒文字、アクセントカラーは1色。
実務的でシンプルなデザイン。
```

## 画像品質チェック

公開前に以下を確認する。

| 項目 | 判定 |
| --- | --- |
| 記事内容と画像が一致しているか | OK / NG |
| 文字がスマホで読めるか | OK / NG |
| 画像容量が10MB未満か | OK / NG |
| AIっぽい意味不明な装飾がないか | OK / NG |
| 図解の流れが本文と一致しているか | OK / NG |
| 有料記事の場合、画像が商品価値を高めているか | OK / NG |
| 著作権・商標・人物画像のリスクがないか | OK / NG |

## 禁止事項

- 本文と関係ない雰囲気画像だけを入れる。
- 文字が小さすぎて読めない図解を入れる。
- 読者を誤認させる収益画像や実績画像を作る。
- 実在人物・企業ロゴ・ブランドを無断で使う。
- AI生成画像を実写の証拠や実績のように見せる。
- 有料記事の価値を画像で水増しする。
