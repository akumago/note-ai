# noteコメント対応ワークフロー

## 方針

コメント返信は完全自動化しない。  
AIは返信案、分類、リスク確認、有料記事や続編への導線案だけを作る。  
最終返信は社長がnote上で手動で行う。

## 理由

- note公式の公開投稿APIがないため、自動返信経路が安定しない。
- コメント欄で機械的な返信を続けるとスパムに見える。
- 読者との信頼形成では、最終判断を人間が持つ方が安全。

## 対応フロー

```text
noteにコメントが来る
  ↓
コメント本文をコピーする
  ↓
GitHub Actionsの Comment reply assistant を手動実行
  ↓
outputs/replies に返信案が出る
  ↓
社長が確認・修正
  ↓
note上で手動返信
  ↓
質問が多いテーマは記事・有料版・FAQ候補へ回す
```

## 分類

- お礼のみ
- 通常質問
- 購入前相談
- 続編リクエスト
- 有料記事への誘導余地あり
- 社長が直接返すべき
- クレーム・炎上・危険対応
- 返信不要

## エスカレーション条件

以下は必ず社長が直接確認する。

- 返金、値引き、契約変更
- クレーム
- 炎上しそうなコメント
- 医療、法律、税務、投資など専門判断
- 個別事情が強い相談
- 個人情報を含む内容
- 有料記事の内容や価格への不満

## 有料導線の扱い

有料記事への誘導は、相手の質問に答えた後に軽く添える。  
売り込みを主目的にしない。

例:

```text
このあたりは、今後チェックリスト形式でもまとめる予定です。
まずはこの記事内の手順だけ試してみてください。
```

## 蓄積ルール

よく来る質問は、以下に回す。

- FAQ記事
- 続編記事
- 有料チェックリスト
- 無料代表作の追記
- プロフィールや固定記事の改善
