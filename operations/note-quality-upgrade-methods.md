# note記事を高品質化する手法集

## 基本方針

note記事の品質は、文章のうまさだけで決まらない。  
読者の絞り込み、一次情報、構成、具体例、図解、読みやすさ、投稿後改善まで含めて品質とする。

AIは本文を量産するのではなく、以下の品質改善を担当する。

## 1. 読者解像度を上げる

悪い例:

```text
noteで稼ぎたい人
```

よい例:

```text
WordPressを1年続けたが月数百円しか稼げず、noteで有料記事を作るべきか迷っている会社員
```

チェック:

- 年齢、状況、悩み、行動直前の迷いが見えるか。
- その人が有料でも買う理由があるか。
- 読後に何をすべきか明確か。

## 2. 読者のBefore / Afterを明確にする

記事を書く前に、読者の変化を定義する。

```text
Before: noteで何を書くか決められない
After: ランキングからネタ候補を5つ出し、1つ採用できる
```

Before / Afterが弱い記事は、読後満足度が下がる。

## 3. 一次情報を入れる

入れるべき一次情報:

- 実体験
- 失敗談
- 検証ログ
- 作業時間
- 数字
- 実際に使ったテンプレート
- 実際に迷った判断
- 変更前 / 変更後
- 読者からの質問

実体験がない場合は、検証ログ、仮説記事、架空ケーススタディとして明示する。

## 4. 具体例を最低3つ入れる

抽象論だけだとAIっぽくなる。  
各記事に最低3つの具体例を入れる。

例:

- 悪い読者設定 / よい読者設定
- 悪いタイトル / よいタイトル
- NG表現 / OK表現
- 撤退すべきテーマ / 継続すべきテーマ

## 5. 反対意見を入れる

信頼される記事は、都合のいい話だけを書かない。

入れる観点:

- この方法が向かない人
- 注意点
- 失敗するパターン
- 例外
- まだ検証できていないこと

## 6. 事実・意見・仮説を分ける

記事内で以下を混ぜない。

- 事実
- 自分の意見
- 現時点の仮説
- 架空ケース
- 引用

表記例:

```text
事実として、note公式は機械的な大量投稿への対応強化を発表しています。
僕の意見としては、AIは投稿係ではなく編集部として使う方が安全です。
現時点の仮説では、有料でも買う人は時間短縮と失敗回避に価値を感じやすいです。
```

## 7. 記事内で自己完結させる

重要な結論を外部LINEや別媒体に隠さない。  
note内で読者が満足できる内容にする。

外部導線を置く場合も、記事本文の価値を削らない。

## 8. タイトルを3案以上作る

制作部は、毎回タイトル案を3-5個出す。

評価軸:

- 誰向けか分かるか。
- 得られる変化が分かるか。
- 煽りすぎていないか。
- 検索・SNSで気になるか。
- noteらしい体験感があるか。

## 9. 冒頭を強くする

冒頭で以下を出す。

- 読者の悩み
- この記事で分かること
- 自分がなぜ書けるのか
- 先に結論

冒頭で一般論を長く書かない。

## 10. 見出しだけで流れが分かるようにする

読者は流し読みする。  
見出しだけ読んでも内容が追えるようにする。

悪い見出し:

```text
メリット
注意点
まとめ
```

よい見出し:

```text
ランキングは真似する場所ではなく、需要を確認する場所
有料でも買う人は「興味がある人」ではない
伸びたらシリーズ化、落ちたら撤退する
```

## 11. 図解を入れる

以下のどれかを入れる。

- フロー図
- 比較表
- チェックリスト
- 判断マトリクス
- ロードマップ
- ケーススタディ図

有料記事では、図解を最低3枚入れる。

## 12. 保存したくなる要素を入れる

保存されやすいもの:

- チェックリスト
- テンプレート
- 手順表
- NG / OK比較
- 週次レビュー表
- 判断基準表

## 13. シリーズ化の余地を作る

記事の最後に、続編にできる論点を残す。

例:

```text
次回は、有料でも買う読者の見分け方を具体的に整理します。
```

ただし、重要な結論を隠して続編に誘導しない。

## 14. 有料化ポイントを設計する

無料記事では考え方を出す。  
有料記事では実行用の素材を出す。

有料化しやすい素材:

- テンプレート
- プロンプト集
- 実例集
- チェックリスト
- 失敗事例
- Before / After
- 週次運用表

## 15. 投稿後に改善する

投稿して終わりにしない。

見る指標:

- ビュー
- スキ率
- コメント
- フォロー増
- 購入数
- SNS反応
- 読了感のあるコメント

改善内容:

- タイトル変更
- 冒頭修正
- 図解追加
- チェックリスト追加
- 続編作成
- 有料版作成
- 撤退判断

## 16. AIっぽさを削る

AIっぽい表現:

- 重要です
- いかがでしたでしょうか
- まとめると
- 多くの人が
- 現代において
- 〇〇することが大切です

改善:

- 自分の判断を書く。
- 具体例を入れる。
- 迷いを書く。
- 失敗を書く。
- 短い文を混ぜる。
- 同じ言い回しを繰り返さない。

## 17. 売り込み臭を減らす

有料導線は自然に置く。

悪い例:

```text
詳しくは有料部分で！
```

よい例:

```text
この記事では考え方まで整理しました。実際に使うネタ選定シートや週次編集会議テンプレートは、別でまとめる予定です。
```

## 18. 権威より検証を出す

肩書きが弱い場合は、検証ログを出す。

```text
3日間ランキングを見た
20個ネタ候補を出した
5つの有料化余地を分類した
```

このような数字は、読者の信頼につながる。

## 19. 読者の行動を1つに絞る

最後にやることを1つだけ書く。

悪い例:

```text
ぜひいろいろ試してみてください。
```

よい例:

```text
今日のnoteランキングから気になるテーマを5つ拾って、そのうち1つだけ「有料でも買う読者」に絞ってください。
```

## 20. 品質スコアを付ける

制作部は公開前に以下を採点する。

| 項目 | 点数 |
| --- | --- |
| 読者が1人に絞れている | 0-5 |
| Before / Afterが明確 | 0-5 |
| 一次情報・検証ログがある | 0-5 |
| 具体例が3つ以上ある | 0-5 |
| 反対意見・注意点がある | 0-5 |
| 図解・チェックリストがある | 0-5 |
| 次アクションが具体的 | 0-5 |
| AIっぽさが少ない | 0-5 |

合計30点未満の記事は公開しない。

## 21. 人間のキャラクター感を出す

noteでは、情報だけでなく「誰が書いているか」が読まれる理由になる。  
AIっぽさを消すためにも、書き手のキャラクター感を意図的に入れる。

入れる要素:

- 口癖
- 迷い
- 本音
- 失敗を笑える余白
- 自分なりの判断基準
- 少しだけ崩した言い回し
- 読者への距離感

入れすぎるとノイズになるため、1記事につき3-5箇所でよい。

### キャラクター感の例

悪い例:

```text
noteで成果を出すためには、ランキング分析を行うことが重要です。
```

よい例:

```text
正直、最初は「ランキング見て真似すればいいんじゃない？」くらいに思ってました。雑ですね。でも、ここが一番危なかった。
```

悪い例:

```text
伸びないテーマは撤退しましょう。
```

よい例:

```text
伸びないテーマを続けるの、けっこう精神にきます。なので、先に撤退ラインを決めておきます。自分を守るルールです。
```

### 入れてよいキャラ要素

- 「最初はこう思っていた」
- 「ここで一回考えを変えた」
- 「これは正直、危ないと思った」
- 「このへん、雑にやると痛い」
- 「ここは割り切る」
- 「自分ならまずこれをやる」

### 避けるキャラ要素

- 過度な自虐
- 内輪ノリ
- 攻撃的な断言
- 読者を見下す表現
- 嘘の武勇伝
- 実績の盛り

### キャラクター設計メモ

今回の運用では、以下のキャラクターを基本にする。

```text
AIを使うが、量産やズルは避ける。
仕組み化が好き。
でも規約や信頼リスクには慎重。
テンションは少し軽め。
実務に落とすのが好き。
伸びたら続ける、落ちたら撤退という割り切りがある。
```

制作部は、記事の最後に「キャラクター感を出した箇所」を3つメモする。
