# note公開後ワークフロー

## 目的

note記事は公開して終わりではなく、反応を見てシリーズ化・有料化・撤退を判断する。

## 公開直後にやること

1. 公開URLを `outputs/metrics/` の対象記事ファイルに記録する。
2. 見出し画像、記事内画像、本文の表示をスマホで確認する。
3. 誤字、画像崩れ、リンク切れを確認する。
4. X投稿案から1-2本だけ投稿する。
5. 過度な連投はしない。

## 24時間後に見ること

- ビュー
- スキ
- コメント
- フォロー増
- SNS反応
- 冒頭で離脱していそうな違和感

## 72時間後に見ること

- スキ率
- コメントや質問の質
- 続編希望の有無
- プロフィールへの導線
- X投稿からの反応

## 7日後に見ること

- シリーズ化するか
- タイトルを変えるか
- 図解を追加するか
- 続編を書くか
- 有料テンプレート化するか
- 撤退または保留にするか

## 改善パターン

| 状況 | 改善 |
| --- | --- |
| ビューが少ない | タイトル、タグ、SNS告知を見直す |
| スキが少ない | 冒頭、読者設定、具体例を見直す |
| コメントがない | 記事末尾に質問を追加する |
| フォローされない | プロフィール導線を追加する |
| 保存されなさそう | チェックリストや図解を追加する |
| 有料化できなさそう | 無料代表作として使い、別テーマへ移る |

## シリーズ化の判断

シリーズ化する条件:

- 読者の反応がある。
- 続編で深掘りできる。
- 有料化につながるテンプレートやチェックリストが作れる。
- 社長の一次情報、検証ログ、仮説を追加できる。

撤退する条件:

- 3本出しても反応が弱い。
- 一般論しか出せなくなる。
- 有料化余地が薄い。
- 読者の悩みが弱い。

## 次に書く記事の決め方

公開7日後に、以下から1つ選ぶ。

```text
反応がよい → 続編
質問が多い → Q&A記事
保存されそう → チェックリスト記事
購入意欲がありそう → 有料テンプレート記事
反応が弱い → 別バズネタへ移動
```
